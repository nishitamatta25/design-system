## Theming and Accessibility

All colors in our system are designed to meet WCAG 2.1 AA contrast requirements. The color system supports both light and dark themes through Tailwind's dark mode functionality.

For dark mode, we generally recommend using higher number variants of colors (e.g., primary-600 instead of primary-500) to maintain proper contrast.

### Color Usage Guidelines

1. Use primary colors for main actions and key UI elements
2. Use secondary and tertiary colors to create visual hierarchy
3. Use semantic colors consistently to convey meaning
4. Use neutral colors for text, backgrounds, and subtle elements
5. Ensure sufficient contrast between text and background colors
6. Test all color combinations in both light and dark themes
